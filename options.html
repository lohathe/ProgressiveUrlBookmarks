Configure your PUB!

PUB (Progressive Urls Bookmarker) is a nice tool, but it needs some settings in order to work properly, to satisfy your needs.

PUB bookmark folder

PUB needs a dedicated bookmarks folder where he can store and manage the bookmarks as he pleases. This way neither you nor PUB have to worry about consequences about your (other) precisous bookmarks!

The default folder is PUB_playground, but if you want to rename it, or you want PUB to use an already existing folder, then please do so!

PUB matching rules

The core feature of PUB is to automatically manage progressing urls. But only you know which urls are to be tracked, and it is you who must tell PUB how to recognize the title and the progressing number from a url. How can you teach PUB? PUB is a lawful-good and low-intelligence creature: he follows your rules and he speaks a very simple language. And the more precise you are, the more useful PUB becomes!

[COMING SOON] If you can supply a rule without the wildcard * then PUB can try to translate a progressing url into the given rule (thus he can try open a content into another site)!

[COMING SOON] You must specify rules telling PUB how the sites you are using format their "progressing url" content. PUB wants to know how to extract the title and episode from an url. An example rule can be: *://my.content.site/*/watch/<title>/episodes/ep-<episode>. You can use the following constraints to express a rule:

  • `*` is a wildcard and matches every character except `/`
  • `<title>` is the placeholder for the content's title. Note that it cannot be directly preceded of followed by `*`. It matches every character except `/`.
  • `<episode>` is the placeholder for the content's progression number. Note that it cannot be directly preceded of followed by `*`. It matches every character except `/`.
  • any explicit character represents an exact match. `/` characters are extremely important since they are used as delimiters.
  • the end of the rule is not strict. This means that PUB will try its best to match as much as possible, and if the start of a url matches a rule but the url is longer, then the exceding part of the url is ignored.

PUB suggestions

You don't remember what you last saw/read? You don't want to open the history/bookmarks to reach your last content? PUB is willing to show you a list of recently bookmarked content.

[COMING SOON] But we all know that if the list is too long or too short then it has little value: you can decide how long this list must be! The default length is 5.
